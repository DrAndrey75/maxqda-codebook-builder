# Security Policy

## Supported Versions
Поддерживается только актуальная версия из последнего релиза.

## Reporting a Vulnerability
Пожалуйста, **не создавайте публичные Issues** для сообщений об уязвимостях.

Предпочтительный способ:
1) GitHub → вкладка **Security** → **Report a vulnerability** (Security Advisory).

Если эта функция недоступна:
1) Создайте Issue с минимальным описанием и без эксплуатационных деталей,
2) Явно пометьте, что это security‑report, и попросите поддерживающих связаться с вами приватно.

## What to Include
- шаги воспроизведения (по возможности),
- ожидаемое и фактическое поведение,
- версия/хеш коммита (или номер релиза),
- влияние и оценка риска.
